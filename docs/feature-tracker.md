# Feature Tracker

This file is a lightweight product snapshot for fast AI and team context.
Keep each section high-level, replace lines instead of expanding them, and avoid detailed implementation notes unless direction changes.

## Markdown
Status: Markdown notes are already a core canvas overlay with raw and hybrid editing, preview mode, tables, checklists, and image support.
Focus: This is usable now for quick capture and semi-structured thinking directly on the canvas.
Next: Keep improving editing polish and prepare it for more reliable AI-driven block-level edits.

## Kanban
Status: Kanban is functional with drag-and-drop cards and columns, search, settings, undo and redo, checklists, and deletion flows.
Focus: The core board mechanics are in place, so the main value now is how well it supports planning inside the broader canvas workflow.
Next: Tighten polish, confirm collaboration behavior, and make AI-assisted board updates feel more natural.

## Web Embed
Status: Web Embed supports pasted URLs, inline viewing, PiP mode, expanded mode, and fallbacks for sites that block iframe rendering.
Focus: It is already useful for bringing outside references into the canvas, but reliability still depends on third-party embed rules.
Next: Improve domain-specific handling, clearer failure states, and stronger support for the embed types we care about most.

## Rich Text
Status: Rich Text is a deeper document surface powered by Lexical, with formatting tools, comments, code and media support, and buffered state syncing.
Focus: This feels more advanced than a quick note and is positioned for heavier writing and review workflows on canvas.
Next: Prioritize stability, collaboration polish, and a clearer boundary between rich notes and full document authoring.

## Prototype
Status: Prototype already has a file-based editing model, live preview, diagnostics, and a dedicated studio route for focused work.
Focus: The foundation is strong for AI-assisted prototyping that can live beside notes, boards, and references.
Next: Improve workflow polish around templates, file operations, persistence expectations, and how prototypes feed back into storytelling.

## Landing Page
Status: The landing page is live in the app and already communicates the broad value of one canvas for rough ideas, research, and structured work.
Focus: It currently works as an early product introduction more than a finalized conversion surface.
Next: Sharpen positioning, align brand and product language, and connect the messaging more directly to standout workflows.

## Dashboard
Status: The dashboard is active and usable for signed-in users, including canvas browsing, search and sort, previews, create, rename, favorite, and delete flows.
Focus: This already reads like a real product surface rather than a placeholder.
Next: Improve organization at scale, empty and loading states, and the feeling of quickly resuming recent work.

## Product Tour
Status: Product Tour currently exists as an experiment-driven canvas tour with guided scenes, overlays, explore mode, and saved layout tooling.
Focus: It demonstrates the product story well, but it still feels closer to a prototype or internal demo than finished onboarding.
Next: Decide whether this becomes a polished user-facing tour, a sales demo surface, or a reusable storytelling system for both.
